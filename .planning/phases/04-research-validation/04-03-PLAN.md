# LangGraph interrupt pattern (per research)
- interrupt(value) raises GraphInterrupt, pauses workflow
- Client receives interrupt payload with review data
- Resume with Command(resume={"decision": "proceed"})
- Node RE-EXECUTES after resume - set all state needed BEFORE interrupt

# Alternative: State-based checkpoint (recommended for MVP)
- Set checkpoint_data before returning
- Client queries state, shows UI, then sends update via API
- Simpler, avoids re-execution issue
- Use this approach: state-based checkpoint

# Workflow steps
1. analyze: Call analyze_niche() with keywords
2. verify: Call verify_demand() with recommendations
3. checkpoint: Set checkpoint_data, await user decision
4. proceed/retry/cancel: Based on user_decision
</context>
