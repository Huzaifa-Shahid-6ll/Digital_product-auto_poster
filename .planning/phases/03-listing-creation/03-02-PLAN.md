  <action>
Create FastAPI routes for listing management (per 03-UI-SPEC.md):

- `POST /api/listings`: Create listing
  - Input: product_id, (optional) title, description, tags overrides
  - Flow: Generate content via AI -> Apply compliance -> Create draft -> Return for review
  - Response: {listing_id, title, description, tags, status: "draft"}

- `GET /api/listings/{listing_id}`: Get listing status and details
  - Response: {listing_id, status, title, etsy_url, created_at}

- `PUT /api/listings/{listing_id}`: Update listing content (edit before publish)
  - Input: title?, description?, tags?, price?

- `POST /api/listings/{listing_id}/publish`: Publish listing (human approval)
  - Flow: Validate -> Upload file (digital) -> Set type=download -> Set state=active
  - Response: {success, etsy_listing_url}

- `DELETE /api/listings/{listing_id}`: Delete/remove listing

Request models:
- `CreateListingRequest`: {product_id, title?, description?, tags?, price_override?}
- `UpdateListingRequest`: {title?, description?, tags?, price?}

Mount in main.py: `app.include_router(listing_router, prefix="/api/listings", tags=["listings"])`
