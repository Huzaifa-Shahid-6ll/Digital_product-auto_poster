## API Design Contract

### Endpoints Required

| Endpoint | Method | Purpose |
|----------|--------|---------|
| `/api/auth/etsy/authorize` | GET | Initiate OAuth flow |
| `/api/auth/etsy/callback` | GET | OAuth callback handler |
| `/api/auth/etsy/shops` | GET | List user's Etsy shops |
| `/api/listings` | POST | Create new listing |
| `/api/listings/{id}` | GET | Get listing status |
| `/api/listings/batch` | POST | Start batch listing |
| `/api/listings/batch/{id}` | GET | Get batch status |

| Pattern | Based On |

| Route structure | `src/api/idea_routes.py`, `src/api/product_routes.py` |
| Request validation | Pydantic models in `src/api/models.py` |
| Retry logic | Per D-04 from Phase 1: exponential backoff |
| Human-in-the-loop | Per D-03 from Phase 2: review before publish |
