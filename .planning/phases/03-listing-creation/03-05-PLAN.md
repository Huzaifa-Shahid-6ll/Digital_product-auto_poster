  <action>
Add batch listing endpoints to existing routes:

- `POST /api/listings/batch`: Start batch listing
  - Input: {product_ids: list[str], stagger_enabled: bool = true}
  - Flow: 
    1. Get approved products from Phase 2
    2. For each product: generate listing -> apply compliance -> create draft
    3. Return first listing for review
  - Response: {batch_id, listings: [{listing_id, product_id, status}], current_index}

- `GET /api/listings/batch/{batch_id}`: Get batch status
  - Response: {batch_id, total, completed, current_index, listings[], next_available_time}

- `POST /api/listings/batch/{batch_id}/approve`: Approve current listing, proceed to next
  - Flow: Publish current -> apply stagger -> wait if needed -> create next draft
  - Response: {success, next_listing}

- `POST /api/listings/batch/{batch_id}/edit`: Edit current listing, then re-submit
  - Input: {title?, description?, tags?, price?}
  - Creates new draft with edits, returns for review

- `DELETE /api/listings/batch/{batch_id}`: Cancel batch

Per D-20: Sequential batch listing - one at a time.
Per D-21: User reviews each before next is created.
